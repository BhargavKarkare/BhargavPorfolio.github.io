Work

Project

Real Estate Applicaiton

The project is on the real estate domain application which works on property listings and deals for home buyers or for real estate agents. where the goal is to make real eastate investing as easy as stock investing.

The application has three layers which are frontend, backend and database, Angular is used for the front-end, Java and Springboot is used for backend and PostgreSQL is used as the database. The architecture includes 12 frontends out of which 11 micro frontends and 1 main frontend. Addtionally there are 18 backend micro services deployed on the EKS cluster. These are connected to a database i.e RDS which has PostgreSQL. All the frontends are deployed on S3 and the main frontend is exposed through Route53 DNS service and integrated with cloudfront for accelerated data transfer. Micro frontends connect to the backend through ingress and internal load balancer which is also integrated with subdomains and exposed using Api.

In the deployment we used 4 branches named as DEV, Test, UAT and Prod and also we has two extra branches named as feature and hot fix branch. The developers works on the dev branch pushing complete code to the test branch for quality checks. Approved code moved to UAT (User Accecptance Test) where end user test the application. Upon approval the code progresses to production. Feature branch is used for adding new features merging back to DEV branch which then follows the rest i.e test, UAT and Production also the HOT-FIX branch addresses the last min changes or immediate bug fiqxes, merging directly with the Prod branch.

The deployment process utilizes various devops tools such as Git for code storage and Jenkins to create multiple CI\CD pipelines and Docker, Kubernetes for micoservice management and deployment. Terraform for provisioning of resources and grafana and prometheus are used for monitoring.

Two CI\CD pipelines were created one for infrastructure deployment and other for application deployment. The infrastructure pipeline are having 3 stages i.e. pull, test and deploy, where the code gets pulled from git undergoes terraform plan to test the code and after the approval the infrastructure gets deployed through terraform apply. Changes in terraform script trigger the pipeline automatically with webhooks.

The application pipeline has 5 stages which are pull, build, test, docker build and deploy the code gets pulled from gitlab with the help of webhooks and build the artifact with the help of maven then tested by using Sonarqube and once it passed the quality gate, Docker image is built using jenkis docker plugin and the application is deployed on the EKS cluster, ensuring the robust and scalable management of microservice.
